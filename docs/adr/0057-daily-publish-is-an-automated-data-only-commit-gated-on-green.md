# The daily publish is an automated data-only commit, gated on green

v3's living seasons are kept fresh by a **season loop**: a scheduled task on a dev machine (pulls are local-only — stats.nba.com blocks cloud IPs, ADR-0010's constraint unchanged) that runs pull session → derives → `hero:sync` → the **full repository gate** (`python -m pytest ingestion -q`, `npm test`, `npm run lint`, `npm run build` — verdict guards and deployed-pair guards included). On green, the loop lands a **data commit** directly on main and pushes: deployed payloads plus their freshness metadata only, with a structured commit message carrying the day's report (new games, headline deltas, gate results). Vercel deploys it. On any failure the publish halts loudly and notifies; nothing ships until a human acts. Code, copy, and configuration never ride the automated commit class — they still travel by PR.

**Why:** daily freshness is the product goal of living seasons, and the human-in-the-loop postures (a human pushing each staged morning commit, or automation stopping at the derived layer) trade that goal for review theater — eyeballing a four-file JSON diff every morning verifies nothing the gate doesn't verify better. ADR-0010's rationale ("which data is live is a reviewed, diffable act, not a build side effect") survives re-grounded rather than weakened: the commit **is** the diffable record, the report in its message is the review artifact, and the full gate is the reviewer — mechanical, and stricter than a human skim. The gate-on-green condition is what makes automation compatible with the guard culture: a verdict claim inverted by the night's games fails a guard, which halts the publish — the halt-and-rewrite morning is an expected operational event (ADR-0059), the system working exactly as ADR-0017 demands.

**Amends ADR-0010, does not supersede it:** the three-layer storage story, the committed deployment copy, and explicit `hero:sync` for completed seasons and manual acts all stand. The data commit is a narrow automated instance of the same explicit sync — same script, same reviewed-and-diffable properties, scheduled instead of typed.

**Consequences:** the automated commit class must stay data-only — a future contributor must not let the loop commit code, copy, config, or guard changes, and must never "fix" a red morning by loosening the assertion that went red (ADR-0014/0017 discipline, now operational). CI on the pushed commit is the second net, not the first: the gate already ran locally. If the dev machine is off or the loop halts, the page serves yesterday's payload — stale-but-exact, disclosed by the frontier metadata (ADR-0058), which is the honest failure mode. A halt notification is part of the loop's contract: a publish that silently stops is a freshness bug, not a quiet day.
