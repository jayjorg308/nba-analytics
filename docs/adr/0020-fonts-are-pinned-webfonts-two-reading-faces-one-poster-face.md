# Fonts are pinned self-hosted webfonts: two reading faces, one poster face

The page's type is pinned to shipped webfonts, not system stacks: **Public Sans** (`--sans`/`--heading`; 400/500/600), **IBM Plex Mono** (`--mono`; 400), and **Big Shoulders Display** (`--display`; 900 only) — self-hosted via `@fontsource` packages imported in `src/main.tsx` and bundled by Vite, with the former system stacks demoted to load-time fallbacks. Reading sizes use exactly the two text faces (sans for prose and UI, mono for figures); the display face exists solely for poster-scale set pieces (today: the hero banner title, ADR-0021) and ships only its 900 cut.

**Why:** the system stacks rendered a different page per OS. The concrete failure: the stat figures' slashed zero — a deliberate 0-vs-O disambiguator in a product that is mostly numbers — existed only where `ui-monospace` happened to resolve to Consolas (Windows); macOS/iOS visitors got SF Mono, unslashed, and never saw the intended page. Self-hosting keeps the deploy static and adds no third-party request (no font-CDN callout). Only the weights actually used ship (400 body / 500 headings and table headers / 600 zone-label names; the mono is single-weight), and shipping the display face solely at 900 is a guardrail, not thrift: with no text weights available it cannot drift into becoming a third reading font — the two-fonts-at-reading-sizes rule enforced at the asset level. Open-license faces (SIL OFL) were chosen over the commercial candidates they stand in for (Neue Haas Grotesk, Druk) so self-hosting carries no license exposure.

**Consequences:** adding a weight means adding a `@fontsource` import in `main.tsx` — `:root`'s `font-synthesis: none` means an unshipped weight falls to the nearest real cut, never a synthesized fake. `--display` must not appear below poster scale, and reading-size type must stay on `--sans`/`--mono`. The pinned families are product identity, not hero configuration — a hero swap changes photo and copy (ADR-0021), never the type system. The ADR-0014 palette guard and ADR-0017 verdict guard are unaffected.
