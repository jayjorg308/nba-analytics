# v1 answers shot selection/making only; shot creation is a v2 thesis

v1's stated question is exactly "is this player taking good shots?" — answered by shot selection and shot making. "How does he create his shots?" (assisted/unassisted, catch-and-shoot vs pull-up, contest/clock context) is a named **v2 thesis**, powered by the deferred Case 2 buckets and Case 3 play-by-play. The tool states the question it actually answers and never implies creation analysis it cannot support.

**Why:** Creation signal lives entirely in deferred data. `shotchartdetail` (Case 1) has none of it — a catch-and-shoot and a pull-up from the same spot are identical dots. Any Case 1 "creation proxy" (e.g. distance as a self-creation hint) is a weak signal masquerading as a strong claim, and would puncture the credibility the whole project is built to protect.

**Consequences:** Drops the more prospect-relevant half of the original two-clause thesis from v1 — a deliberate narrowing, not an abandonment. Creation is *scheduled* (v2 inherits a crisp charter) rather than dropped. Standing guardrail: a future contributor must not "helpfully" add a Case 1 creation proxy; doing so violates this decision.
