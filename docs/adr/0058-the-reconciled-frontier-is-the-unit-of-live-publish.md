# The reconciled frontier is the unit of live publish

A living season never publishes "whatever this morning's pulls returned." The unit of publish is the season **through the reconciled frontier**: the latest game date at which every source for the hero-season is exactly coherent. All five eligibility gates and every cross-payload identity are asserted exactly *through the frontier* — no gate weakens, no tolerance is introduced; what becomes movable is the as-of point the exactness is claimed for. A source lagging **beyond** the frontier (last night's play-by-play pair not yet published upstream) defers that game to the next pull session — the loop publishes through the previous coherent date, and that is a normal morning, not a failure. Any incoherence **at or inside** the frontier (sources disagreeing about a game the corpus already covers) is grammar or source drift and hard-halts the publish for a human, exactly as ADR-0044/0046/0054 intend.

**Why:** the sources move on different clocks — league totals update continuously, the shot and tracking endpoints are cumulative as-of-now, play-by-play appears per game with lag — so a morning where the totals endpoint counts a game whose play-by-play pair doesn't exist yet is routine, not exceptional. Without the frontier the options are all bad: a tolerance ("within one game") is forbidden by the exactness culture; all-or-nothing mornings let a routine upstream lag silently cost a day of freshness; publishing per-payload whatever derives clean ships siblings that contradict each other, which the reconciliation battery exists to prevent. The frontier keeps every identity byte-exact while surviving asynchrony daily.

**Frontier-anchored pulls are the mechanism.** The cumulative sources have no per-game grain — a tracking dashboard or league-totals response cannot be sliced by date after the fact — so the frontier must be enforced at pull time. A pull session runs play-by-play/box acquisition first (their availability fixes the frontier), then pulls every cumulative source with the frontier as its date ceiling (`DateTo`). The raw artifact records its date range, which CONTEXT.md's raw-artifact definition always required ("pull-date and games-included (or date-range)"). Same-day retries append timestamped snapshots — append-only holds (ADR-0006). The endpoint-behavior spike is a build precondition, not a formality: for each cumulative source, a `DateTo` pull of a completed season must reconcile exactly against the date-filtered rows of the full snapshot (shots), against the date-filtered shot count via the General-family identity (tracking), and the `LeagueAverages` frame's semantics under `DateTo` must be established. If an endpoint fails the spike, the frontier rule does not bend — the pull design for that source changes.

**The frontier is contract metadata.** Every payload's `_meta` gains `dataThrough` and `gamesIncluded`, stamped at derive from the pull session — a coordinated schema bump across all four contracts, schemas + derives + regenerated goldens moving together per the cross-language convention. **Four-way frontier equality joins the cross-payload reconciliation battery** (derive-time and deployed-pair guards): a one-sided sync now fails visibly as a frontier mismatch, strengthening the existing guard. Completed seasons carry the same fields (final game date, full game count) — one payload class, no special casing. On a living season, Gate 5's season-total oracle becomes a frontier-total oracle: reconstructed free throws through the frontier must equal the frontier-anchored totals artifact exactly, and a failure still names the missing game (ADR-0054's diagnostic path unchanged).

**Consequences:** the published page is always exactly reconciled, at worst a few hours behind the real world — stale-but-exact is the designed failure mode, disclosed by the through-date the UI renders (ADR-0059's structural framing). The replay proof falls out of the same mechanism: the season loop run against a *completed* season over a calendar of simulated frontier dates is real data through real endpoints, no synthetic fixtures, with a terminal oracle no simulation could offer — the final replay frame must equal the known completed-season payloads exactly. A future contributor must not introduce a proximity tolerance anywhere in frontier computation, must not publish sibling payloads whose frontiers disagree, must not slice a cumulative source client-side as a substitute for a frontier-anchored pull, and must not treat "the frontier is yesterday" as an error state — deferral is the design.
