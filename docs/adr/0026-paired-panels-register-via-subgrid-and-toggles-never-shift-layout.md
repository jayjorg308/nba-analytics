# Paired panels register via subgrid, and toggles never shift layout

Content that reads as one comparison aligns through CSS subgrid over shared parent tracks, not by hoping copy stays the same length: the headline pair pins its cards' header/numbers/note rows to shared tracks, the chart and zone panels pin the toggle+legend block and the table's section header to one shared track (court top = table top at any width), and the headline number columns are equal thirds so both cards' stats land at the same x. The companion invariant: switching a display mode (Zones/Shots) never changes the page's geometry — both court legends stay mounted, stacked in one cell with the inactive layer `visibility: hidden`, sized near-equal (the scale's boundary values render inside the color bar so the legend is two compact lines), asserted by the ChartPanel tests.

**Why:** the pair-reads-as-one-comparison argument (ADR-0016, ADR-0018) is carried visually by register — numbers beside numbers, notes beside notes, deltas in the same spot on both cards; flex packing by content width drifts out of register whenever copy changes. A toggle is view state; a page that jumps ~20px on a view switch reads as broken chrome, and reserving the tall legend's height without equalizing the legends just trades the jump for a dead band.

**Consequences:** the zone table's caption lives OUTSIDE the `<table>` (a `<caption>` cannot leave its table to occupy a shared track) and is wired back with `aria-labelledby`; its description also stays outside the `h2` so the heading's accessible name never collides with the SHOT MAKING card in role queries. A panel that becomes a subgrid stops being a scroll container — the overflow wrapper and `min-width: 0` chain inside it are what keep the table scrolling internally, because the page must never scroll horizontally. Legend layers must be *hidden*, never unmounted — the ChartPanel tests encode this; "cleaning up" the hidden layer reintroduces the jump.
