# The hero banner carries the question as poster-scale hero config

The page opens with a **hero banner**: a black-and-white action photo of the hero player with the thesis question set as poster type — the `h1` itself lives on the photo (kicker · question · "↓ the verdict" cue), so the ADR-0018 argument order (question → verdict → evidence) is preserved, only louder. The banner's *content* is authored hero configuration exactly as the verdict is (`heroConfig.hero`: image URL, alt text, per-layout focal points, kicker); its *treatment* is product: the B&W is a CSS `grayscale` filter over a color asset, and the committed image is a web-sized JPEG derivative (2048px, ~570 KB) — full-resolution sources stay out of the repo. Two layouts: wide (>1024px) sets the photo as a right-anchored 60% panel whose left edge mask-fades into the dark ground, with the title vertically centered beside it; narrow sets a full-bleed 3:4 poster with the title bottom-anchored.

**Why:** ADR-0018's differentiation stance, extended to the page's first pixels — generic stats sites open with chrome and a chart; this page opens like an editorial feature, and the chosen photo argues the thesis literally (a dunk at the rim is the shot diet the verdict describes). The `h1` lives *on* the banner rather than below it because the alternative — a decorative banner above a repeated title — would put the question on the page twice and demote the poster to wallpaper. Authored config because photo, focal points, and kicker are hero copy in the same sense the verdict is (ADR-0017): swapping heroes must not mean redesigning the banner. B&W-by-filter keeps the treatment tunable per photo and the asset reusable in color. The overlay ink is white in both themes deliberately: the banner is its own dark surface with scrims guaranteeing the ground — outside the ADR-0014 palette contract, which governs the making scale, not the poster.

**Consequences:** a hero swap now includes a photo: `heroConfig.hero` needs a new image, alt text, focal points for both layouts, and kicker. Commit only web-sized derivatives, never full-resolution sources (the first source PNG was 26 MB — repo history is forever). Don't move the `h1` back below the photo or duplicate the question in page copy. The wide layout's title cap (72px) and panel width (60%) are a tuned pair — at 80px the title's tail collided with the player's white jersey — so retune them together when the photo changes. The narrow (≤1024px) full-bleed poster is the approved reference experience; wide-layout adjustments must not leak into it.
