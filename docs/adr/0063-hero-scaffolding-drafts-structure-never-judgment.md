# Hero scaffolding drafts structure, never judgment

`npm run hero:scaffold -- <slug> <season>` generates a **season argument's** mechanical skeleton — the unit is ADR-0060's, deliberately: a new hero module when `src/heroes/<slug>.ts` is absent, a `seasons[]` append when it exists, plus the season's guard file and the registry entry. Mechanical fields get real values: the slug (user-chosen — it is the URL), the player name read from the shot payload's `_meta.player` (never a CLI arg; the payload is authoritative and typo-proof), the verbatim thesis formula (ADR-0005), the season string, the conventional banner path `img/<slug>-hero.jpg`, the conventional headshot path `img/<slug>-headshot.png` (ADR-0065 — a mechanical asset, so the emitted comment carries the CDN download formula), and on a new module `canonicalSeason` = the scaffolded season. Every authored field — verdict, kicker's team/number, image alt, both focal points — is emitted as a `TODO(scaffold)` sentinel; `teamLogoPath` is a commented-out affordance pointing at the asset-interface test. The guard skeleton is structure only: the explicit `seasonArgumentOf` pin, the payload-path/skipIf/parse/aggregate preamble, **empty** claim arrays for every claim family the lexicon currently ships, the universal tripwire test verbatim, the claim-mapping comment header as a TODO template, and one commented-out example claim. No thresholds, no directional assertions, nothing read from the payloads, no drafted prose — ever. Flip-PR bookkeeping (the `canonicalSeason` move, `season.config.json`, vocabulary graduation) stays human.

Guards go **per-season files** — `<slug>.<season>.guard.test.ts` — and the four existing `<slug>.guard.test.ts` files migrate by rename, content unchanged except adopting the shared authoring helper. An **authoring tripwire** rides in every guard: one shared pure helper (the lexicon's pattern — pure functions returning problem lists) asserts no sentinel remains in any authored field and every referenced image asset exists under `public/` (banner, headshot, team mark); it runs outside the payload `skipIf`, so it fails on clean clones and CI alike. The tool registers the hero eagerly (append at the end of `HEROES`), hard-fails unless all four derived payloads resolve for the slug/season (the `hero:sync` partial-fails stance, applied at the front door), and never overwrites anything that exists — an existing module means append mode, an existing season entry or guard file is a hard fail, and there is no force flag. The generator is a pure emit core in `src/scaffold/` behind a thin `scripts/hero-scaffold.ts` CLI (the `hero:report` pattern), tested by structural unit tests plus one temp-dir integration test that dynamic-imports the emitted module; deliberately **no byte-golden** of generated code.

**Why:** the one boundary this codebase defends hardest is ADR-0017's — the verdict is authored, never computed — and a generator that emits candidate prose (even marked "draft") anchors the author and blurs it. The same logic kills machine-proposed claims: claims are named after the verdict words they back, the verdict does not exist at scaffold time, and pre-made assertions would invert the authoring flow (report → verdict → claims) and tempt prose written to fit them; CONTEXT.md is already explicit that claim headroom is an authoring input only, never a guard input. What the tool automates instead is transcription and boilerplate — and *completeness enforcement*, which review conventions cannot provide: before this ADR, a scaffolded-synced-registered hero with literal `TODO` copy shipped green (the copy tests only guard em dashes), and nothing anywhere asserted a banner image exists. The tripwire converts both from reviewable to impossible, and eager registration converts "forgot the registry entry" (a silently dead, fully-green module) into "everything red until the add is complete" — the house pattern of loud incompleteness. Per-season guard files make the code layout mirror the domain model (a hero is a directory of season arguments) and give ADR-0060/0061's freeze discipline a physical form: "the prior argument stays frozen verbatim" becomes "that file never changes again," auditable in git history, instead of a property of part of a shared file. Uniform migration beats grandfathering because two coexisting conventions cost every future reader and the tool a branch, forever. No byte-golden because goldens guard cross-language data contracts where byte-stability is the point; nothing parses the emitted bytes, structural invariants are what matter, and every real scaffold must survive `tsc` plus the full suite in the PR that uses it — the gate is the ultimate verifier.

**Consequences:** registration moves from the recipe's last step to its first, and the repo is deliberately red between scaffold and finished authoring — a half-done add can never merge, which is what makes the tool safe to run casually. The migrated guards gain the asset-existence check, so the broken-banner hole closes for the existing four heroes, not just future ones. The emit core is code in the repo, not a frozen template: when the lexicon graduates a vocabulary family (growth, at the first flip with a prior argued season), the skeleton's claim-family wiring is updated with it. The scaffolder makes Peterson's born-live add (ADR-0059's stretch) a scaffold-plus-authoring morning, and every future flip PR starts from a generated season append instead of hand-copied boilerplate — while the flip's judgment moves stay exactly as manual as ADR-0059/0060 designed them. CLAUDE.md's guard-naming references update in the build PR that performs the renames.

**Amendment (2026-07-23, the build):** two closures from the end-to-end proof (a real scaffold run against staged copies of real payloads). First, a vitest `describe.skipIf(true)` still executes its factory at collection time, so a guard that loads payloads at describe scope crashes with ENOENT instead of skipping — and a scaffolded guard's deployed payloads are legitimately absent until `hero:sync` runs, which would have buried the authoring tripwire's designed failure under a collection error. The emitted skeleton therefore loads its metrics conditionally outside the describe (existence-checked, `null` when undeployed) — the one structural place it deliberately differs from the four hand-written guards, whose committed payloads make the latent crash unreachable. Second, the structured edits (the seasons[] append and the registry registration) detect and preserve each file's own line-ending convention: a CRLF checkout must neither miss LF-only anchors nor grow mixed-endings files.
