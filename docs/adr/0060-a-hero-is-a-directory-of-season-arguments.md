# A hero is a directory of season arguments and /<slug> is the canonical alias

The hero-season becomes the unit of a page. Every **season argument** — a hero-season shipped complete: authored verdict, colocated guard claims, all four payloads — lives at a stable **season permalink** (`/<slug>/<season>`) from the day it ships, and `/<slug>` is the **canonical alias**: it renders the hero's **canonical season** in place (never a redirect; static hosting has no server to redirect with). The registry keeps hero identity in one module (slug, player name, thesis, banner image/positions/team mark) and grows an ordered `seasons[]` of season-owned copy — season, verdict, kicker (which already embeds the season string) — plus an explicit `canonicalSeason` field. The live flip (ADR-0059) stops replacing an argument and starts moving the pointer: the flip PR moves `canonicalSeason`, and the prior season argument survives at its permalink, gaining only a quiet structural forward link to the current season. The hero index stays one poster tile per hero, linking the canonical alias.

**Why:** ADR-0059's flip as originally written *replaces* a hero's completed argument — Ace Bailey's rookie-season argument would vanish the day his page flips to living 2026-27 (projected late November). Preserving it forced the page-unit question, and per-season pages is the answer ADR-0022's grammar already implies: complete arguments at real URLs, navigated by plain links, never a switcher. The alternatives each break something standing. One two-season page forks the one-class-of-hero-page rule (ADR-0030) and pushes every act toward dashboard density. A comparison-only page demotes the rookie argument from a shareable standalone page to embedded evidence. Seasoned URLs for prior seasons only means a season's URL changes meaning exactly at the flip — links shared during the season would silently show a different season afterward. Flat suffixed slugs (`/ace-bailey-2025-26`) encode the hero–season relation in string convention instead of structure. A separate archive list beside the registry forks ADR-0022's single source of truth. A tile per season turns the directory of arguments into a season archive.

**Consequences:** every registry consumer — the router, the index tiles, `hero:sync`, the four real-data guard suites, the identity and punctuation guards — iterates hero × seasons; a single-season hero is the one-element case, so there is still exactly one class of hero page. `routeSlug` learns to split one slash (the SPA fallback already rewrites deep paths); `document.title` carries the rendered season; the kicker moves into the season entry as season-owned copy. A flip PR's full contents become: the authored live verdict + claim mapping, the `canonicalSeason` move, the dark→live config flip, the prior page's forward link, and — when a prior argued season exists — optionally the first growth-sentence (ADR-0061). Nothing else on the prior page changes. ADR-0022 is extended, not superseded; ADR-0059's "replaced by its own flip PR" is amended to "superseded as canonical by its own flip PR."
