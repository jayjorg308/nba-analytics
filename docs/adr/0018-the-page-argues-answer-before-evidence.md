# The page argues: answer before evidence

The hero page is structured as an argument, not a reference: question (title) → verdict in words (ADR-0017) → the two-axis answer as paired headline blocks with equal billing (ADR-0016) → the zone-shaded court as the default view with its legend read first → a zone table ordered payoff-first. Every generic stats site leads with the chart and table and lets the reader draw conclusions; this tool leads with the conclusion and lets the chart and table back it up.

**Why:** the shot chart is a commodity — a made/missed scatter looks like every other site's and invites the reader to pattern-match the tool to them. The differentiation is the stated, guarded verdict and the two-axis decomposition behind it, so prominence follows differentiation. Several individually small choices only make sense together under this stance, and are deliberate: the raw scatter is the *secondary* court view (the zone-shaded court is the argument painted on the floor); the table has **no FG% column** (PPS is the unit of shot quality — ADR-0001 — and Making Δ already encodes FG%-vs-league; raw FG% lives on the court's zone detail card — ADR-0027); the table's reference column (PPS lg) trails so any overflow cuts reference material, never the payoff; and FGA stays visible everywhere as the honesty anchor that makes the † flag interpretable — never hover-hidden (hover is invisible to touch).

**Consequences:** do not regress these piecemeal — re-adding an FG% column, defaulting the court to Shots, or moving the verdict below the evidence each individually looks like a harmless familiarity fix and each dismantles the argument structure. The evidence still has to be complete and honest (flags, muted-never-deleted rows, reported backcourt) — this ADR changes the *order of prominence*, never what is disclosed. When v2 adds creation context, it extends the verdict-first structure (the verdict grows a "why", per ADR-0005's boundary) rather than reverting to a dashboard of charts.
