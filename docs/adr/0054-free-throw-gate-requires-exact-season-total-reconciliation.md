# The free-throw gate (Gate 5) requires exact season-total reconciliation

Hero eligibility gains a fifth gate: the hero-season's reconstructed free-throw record must reconcile **exactly** with the hero's season FTM/FTA from the league season-totals artifact — a verbatim `LeagueDashPlayerStats` (Totals) snapshot stored append-only at `data/raw/_league/<season>/totals/<pull-date>.json`. The same artifact serves double duty: gate oracle (hero rows) and league free-throw baseline source (all rows summed per ADR-0004), so the gate adds one pull, not two.

**Why the gate exists:** the play-by-play corpus is keyed off games containing a hero *shot* (Gate 4, ADR-0044), but free throws do not require a field-goal attempt — a hero could shoot free throws in a game the corpus never pulled, and the per-game oracles would pass while the season silently undercounted. Season-total equality converts that hole from luck into structure: if every game reconciles and the season total matches an independent source, no free throw happened outside the corpus. All three current heroes pass exactly (84/119, 337/378, 540/614; Cody's five shotless games contained zero free throws — now a proven fact, not an assumption).

**The game universe stays shot-keyed.** Re-keying `pull_play_by_play.py` on a per-hero game log was considered and declined: it adds an endpoint dependency and a pull-script rework for a condition never yet observed. When the gate does fail, it *names the gap*: the season-total delta localizes to a game (via `PlayerGameLogs` as a diagnostic tool, never a second source), and the remedy is pulling that game's play-by-play/box-score pair — the pull script accepts explicit game IDs for exactly this. A gate failure mid-derive is the system working, not breaking.

**Consequences:** like Gate 4, the gate is binary at the season level and enforced as a derive-time hard fail — a free throw payload is never persisted for a season that cannot prove completeness. CONTEXT.md's hero-eligibility entry now reads five gates. In-progress-season behavior (a live season's totals move between pulls) belongs to v3's freshness machinery and must not weaken the completed-season gate, exactly as ADR-0044 held for Gate 4.
