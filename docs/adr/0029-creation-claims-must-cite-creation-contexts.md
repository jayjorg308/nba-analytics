# Creation claims must cite creation contexts; ADR-0005's absolute boundary becomes conditional

v2.0 replaces ADR-0005's blanket prohibition with a conditional one: product copy may make a creation claim **iff** the claim is backed by Case 2 creation contexts — the tracking dashboards' pre-aggregated buckets (ADR-0030). Case 1 inference stays forbidden exactly as before: `ACTION_TYPE` remains quarantined in the raw layer, and no creation signal may ever be derived from `shotchartdetail`.

**Why:** ADR-0005's boundary was never "creation is unspeakable" — it was "don't claim what the data can't back," with v2 scheduled to bring the data. The bucket-grain payload brings it: a catch-and-shoot/pull-up split from the tracking dashboards is a real measurement, not a Case 1 proxy, so a verdict sentence citing it clears the honesty bar the boundary exists to protect. The relaxation is conditional rather than total because the failure mode ADR-0005 named — a weak signal masquerading as a strong claim — is still live for Case 1 data; only the *source* of a creation claim changed, not the standard.

**Enforcement (the tripwire flip):** the per-hero verdict guards (ADR-0017/0022) declare typed claims — `selection` / `making` / `creation` — through a shared helper. The lexical tripwire flips from "creation vocabulary is forbidden" to "creation vocabulary requires ≥1 creation-kind claim asserted against the creation metrics" (`aggregateCreationMetrics` output, ADR-0030). Sentence-by-sentence semantic correspondence stays authored discipline, exactly as it already is for selection/making claims — the helper makes it impossible to say "pull-up" in a verdict with zero bucket-backed assertions behind it, not to verify prose meaning. The flip lands in the same PR as the first creation why-sentence, so there is never a window where creation vocabulary renders unguarded.

**Consequences:** the verdict grows its "why" sentence (the extension ADR-0018 anticipated), and it is **required** for every registered hero — one class of hero page, per ADR-0030's required-payload rule. ADR-0005 is not wholly superseded: its Case 1 quarantine and its "no creation proxy from Case 1" guardrail remain in force verbatim; only the absolute lexical boundary is replaced by the backed-claim rule. A future contributor must not read this ADR as permission to infer creation from shot-chart rows — the permitted citations are Case 2 contexts today and Case 3 reconstruction when v2.5 earns it.
