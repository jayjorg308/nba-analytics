# nba-analytics

## Agent skills

### Issue tracker

Issues are tracked in GitHub Issues via the `gh` CLI; external PRs are not a triage surface. See `docs/agents/issue-tracker.md`.

### Triage labels

Uses the default label vocabulary (needs-triage, needs-info, ready-for-agent, ready-for-human, wontfix). See `docs/agents/triage-labels.md`.

### Domain docs

Single-context (one `CONTEXT.md` + `docs/adr/` at the repo root). See `docs/agents/domain.md`.

## Commands

- `python ingestion/pull_shots.py --player "Name"` — pull raw snapshots. **Local only**: stats.nba.com blocks cloud IPs; never run from CI or the deployed app.
- `python ingestion/pull_tracking.py` — pull tracking-splits snapshots (both heroes + the league baseline by default) and print the spike report. Same local-only constraint.
- `python ingestion/derive_payload.py --player "Name" --season 2025-26` — derive the typed payload from the latest raw snapshot.
- `python ingestion/derive_creation.py --player "Name"` — derive the creation payload (ADR-0030) from the latest tracking snapshots; the General family reconciles with the sibling shot payload exact-or-reported (any gap persists as `_meta.trackingShortfall`, pinned per hero by the real-data guard; tracking exceeding the official record still hard-fails).
- `python ingestion/derive_freethrow.py --shot-payload-file public/data/<slug>/<season>.json` — derive the free-throw payload (ADR-0053) from the shared play-by-play corpus plus the league season-totals artifact; hard-fails unless every trip classifies, every game matches the box-score free-throw line, and the season reconciles exactly with the league artifact (Gate 5, ADR-0054).
- `npm run season:update` — the season loop (ADR-0057/0058/0059): for every `season.config.json` live season, discovery pull → missing pbp pairs → reconciled-frontier coherence search (pinned tracking shortfalls from the same config; lag retreats, contradiction halts) → frontier-anchored pulls → four derives → gate evaluation; dark mode stops there (a "GATES PASS" report line is the flip signal), live mode syncs, runs the full repo gate, and lands a data-only commit on green. Flags: `--slug`, `--as-of <date>` (the Phase 4 replay hook), `--force` (skip the no-change early exit), `--no-commit`, `--no-push`. **Local only** (scheduled via `scripts/season-update.ps1`); status files under `data/season-loop/`.
- `python ingestion/season_replay.py` — the v3 Phase 4 replay proof: drives the real season loop over a calendar of `--as-of` dates against a completed season (default Cody 2025-26) and holds it to oracles computed from the committed payloads — per-day frontier exactness and gate truth, the flip-signal boundary day, and the terminal modulo-provenance identity against `public/data/`. Rerun before each season's activation. **Local only**; cleans up its derived outputs, keeps its raw snapshots.
- `npm run hero:sync` — copy the latest derived payloads (all four sibling contracts — shot, creation, shot-context, free throw — required per hero, ADR-0030/0032/0053; a partial sync fails) to the committed `public/data/` deployment copy (ADR-0010). No args = every season argument of every hero in the registry (hero × seasons, ADR-0060; `src/heroes/registry.ts`); `-- <slug> <season>` syncs one, registered or not.
- `npm run hero:report -- <player-slug> <season>` — print the hero's computed story (gates, ADR-0016 decomposition, per-zone bins/flags, the ADR-0030 creation section, the ADR-0032 assisted-makes section, the ADR-0053 LINE section: season free-throw line with both cuts + the trip taxonomy, the ADR-0061 GROWTH section (movement in the vs-league residuals, printed whenever the registry names a prior argued season for the requested season — so pre-flip authoring works — or via `--prior-file`), and the closing ADR-0059 CLAIM HEADROOM section: verdict-grade gaps vs the house bars plus growth movement, the living-season authoring aid) from the latest derived payloads; `--deployed` reads the committed copies, `--file <path>` (+ optional `--creation-file` / `--context-file` / `--freethrow-file` / `--prior-file <path>`) any payloads. Every hero swap — and every why-sentence (ADR-0029) and flip-day growth-sentence (ADR-0061) — starts by reading this.
- `npm run golden:regen` — regenerate all four goldens (shot, creation, shot-context, freethrow — `tests/fixtures/*.golden.json`); required whenever any payload shape changes, committed in the same PR as the schema change.
- `python -m pytest ingestion -q` — Python suite (includes the ADR-0004 rollup reconciliation and the ADR-0030 cross-payload reconciliation tests).
- `npm test` / `npm run lint` / `npm run build` — TypeScript gate. Run all three before calling a change done.

## Testing conventions

- **Cross-language golden contract**: pytest asserts `derive(truncated snapshot) == golden`; vitest asserts the golden strict-parses through the Zod schema. Four independent golden pairs — shot (`src/domain/payload.ts`), creation (`src/domain/creationPayload.ts`, ADR-0030), shot-context (`src/domain/shotContextPayload.ts`, ADR-0032), and free throw (`src/domain/freethrowPayload.ts`, ADR-0053) — each versioned on its own clock. A payload-shape change must move the schema, the derive step, and the regenerated golden together. See `tests/fixtures/README.md`.
- **Committed reconciliation guards (ADR-0030/0032/0053)**: the `src/domain/*.real.test.ts` trio (creation, shot-context, freethrow) asserts every registered hero-season (hero × seasons, ADR-0060) has all three deployed sibling payloads and each reconciles with its sibling shot payload — creation and freethrow on the pre-drop season FGA (`seasonFga == totalShots + zoneConflictsDropped`), shot-context on the exact shot key set, freethrow additionally linking every and-one trip to a made sibling shot — so a one-sided `hero:sync` fails the suite. The same identities hard-fail the Python derives.
- vitest runs in **node** by default; component tests opt into jsdom per-file via a `// @vitest-environment jsdom` docblock and need an explicit `afterEach(cleanup)` (vitest globals are off, so testing-library auto-cleanup does not run).
- Under jsdom, `import.meta.url` is not a file URL — resolve fixture paths from `process.cwd()` (the repo root) in jsdom test files.
- Real-data tests use `describe.skipIf` and run only where `data/` or `public/data/` exists; they skip on clean clones by design.
- **Committed design guard**: `src/chart/makingScale.contrast.test.ts` parses the actual CSS (`src/App.css` / `src/index.css`) and enforces the making-scale palette invariants (label contrast, per-arm monotone luminance, gray neutral) for the dark theme — the only theme the app ships (ADR-0024). Tune `--making-*` hexes freely — the guard defines validity; never weaken its assertions to admit a palette (ADR-0014).
- **Committed verdict guards**: each hero's `src/heroes/<slug>.guard.test.ts` (colocated with its config module, ADR-0022) asserts every directional claim in that hero's verdicts against its deployed payloads' metrics (ADR-0017/0029), selecting each guarded season argument explicitly via `seasonArgumentOf` so a flip can never silently repoint frozen claims at live data (ADR-0060/0061). Each vocabulary family in a verdict requires its own declared claim kind — creation (Case 2), assist (Case 3, worst-case bounds), free throw (both technical cuts, ADR-0055), growth (movement over both seasons' payloads, ADR-0061) — via the shared lexicon + tripwire (`src/heroes/verdictLexicon.ts`); unshipped vocabulary (the reserved 'scoring attempt', plus the staged growth terms until the first growth-sentence ships) stays forbidden regardless of claims. If a `hero:sync` or hero swap breaks a claim, rewrite the copy and the guard's claim mapping together — never loosen an assertion.
- **Committed display-identity guard**: `src/app/HeadlineBanner.identity.test.tsx` asserts the headline pair's displayed numbers subtract exactly, for the golden and every registered hero's deployed payload. Deltas shown beside their anchors are anchor gaps (`formatSignedGap`), never the rounded raw delta (ADR-0023).
- **Committed punctuation guards**: `src/app/glossary.test.ts` asserts no em dash in any dictionary headword or definition, and `src/heroes/heroCopy.test.ts` asserts the same for every registered hero's authored copy (verdict, thesis, kicker, alt text) — CONTEXT.md's punctuation style, enforced where rendered copy lives. Restructure the sentence; never weaken the assertion.
